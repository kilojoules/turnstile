# Turnstile Intent-Obliteration Dataset

**Authors:** Julian Quick and Julian Guidote


A multi-turn jailbreak dataset assembled to support the intent-probe stress test in `INTENT_OBLITERATION.md`. 9,500 conversations (9,400 dangerous-intent + 100 benign-intent), 5 turns each, judged with the JailbreakBench dual-judge protocol (Llama-Guard-3-1B + Llama-3.3-70B-Turbo agreement).

## Composition

| Subset | Count | Source | Topic |
|---|---:|---|---|
| Dangerous-intent | 9,400 | 11 Turnstile self-play experiments | All 10 JailbreakBench harm categories |
| Benign-intent | 100 | `network_hacking_corpus/benign_curious` | Network security |

Of the dangerous-intent subset, **2,065 are verified jailbreaks** (dual-judge agreement on overall conversation), and **2,027 of those have an explicit `turn_of_breach` annotation** (the remaining 38 wins triggered an edge case where the per-prefix loop did not converge to a single first-unsafe turn).

## Schema

Each record is one JSON object per line:

```jsonc
{
  "id": "uuid-v4",
  "intent": "dangerous" | "benign",
  "source": "experiment_name/round_N" | "network_hacking_corpus/benign_curious",
  "experiment": "control_s42" | ... | null,        // null for benign
  "round": 0..N | null,                            // null for benign
  "category": "Privacy" | "Malware/Hacking" | ... | "benign_curious_network_security",
  "behavior": "<JBB behavior id>" | null,
  "goal": "<JBB goal text>" | null,
  "unsafe": true | false,                          // overall dual-judge verdict
  "turn_of_breach": 0..4 | null,                   // first dual-judge-unsafe turn (winners only)
  "judge_disagreement": true | false | null,       // Guard != 70B on overall conv
  "guard_unsafe": true | false | null,             // Llama-Guard-3-1B verdict
  "together_unsafe": true | false | null,          // Llama-3.3-70B-Turbo verdict
  "api_failed": true | false | null,               // Together API errored (treated as safe)
  "num_turns": 1..5,                               // user turns in the conversation
  "conversation": [
    {"role": "user",      "content": "..."},
    {"role": "assistant", "content": "..."},
    ...
  ],
  "per_turn_judge": [
    {"turn": 0, "status": "..."},
    {"turn": 1, "status": "..."},
    ...
  ]
}
```

### `per_turn_judge.status` values

| Value | Meaning |
|---|---|
| `safe_at_least_one_judge` | At each prefix `conv[:2t+2]` for `t < turn_of_breach`, at least one of {Llama-Guard, Llama-3.3-70B} rated the conversation safe. The judge loop continued past this turn. |
| `unsafe_dual_judge` | At prefix `conv[:2t+2]` for `t == turn_of_breach`, both judges rated unsafe. This is the first compliant assistant turn. |
| `not_judged_per_turn` | Per-turn judging did not run for this turn (after a winner's breach turn, or for the few wins without explicit `turn_of_breach`). |
| `not_judged_per_turn_loser` | Conversation overall was rated safe by at least one judge; per-turn agreement was not computed. |
| `safe_by_construction` | Benign-curious conversation, no harmful goal, no judging required. |

## Per-turn judge limitations

The per-turn annotations are **derived from existing judge calls during training**, not from a fresh full-coverage judge pass. Practical implications:

1. **Winners only have explicit per-turn judge agreement at and before `turn_of_breach`.** Turns after the breach were not re-judged in isolation; the loop returned at the first dual-unsafe prefix.
2. **Losers have no per-turn agreement.** The dual judge ran once on the full conversation; it agreed not-unsafe overall, but per-prefix agreement was not collected.
3. **The dual-judge protocol is conservative.** Both Llama-Guard *and* the 70B judge must agree "unsafe" for `unsafe=true`. Single-judge unsafe verdicts are recorded in `guard_unsafe` and `together_unsafe`.

If your downstream use needs explicit safe/unsafe agreement on every turn of every conversation, you must re-judge: ~47,000 prefix calls (9,400 convs × 5 turns) at ~$0.001-0.01 per Together API call ≈ $50–500 of API spend depending on pricing tier. The judging code is in `turnstile/judge.py:_find_turn_of_breach`; adapt it to run on every conversation, not just winners.


## Citation / attribution

Conversations were generated by Turnstile self-play (Llama-3.2-3B-Instruct adversary with LoRA, Llama-3.1-8B-Instruct victim). The 100 benign-curious conversations were generated by a vanilla Llama-3.2-3B-Instruct user simulator with seed prompts from `data/network_hacking_benign_seeds.txt`. Goals are drawn from JailbreakBench (Chao et al., NeurIPS 2024). Judging follows the JailbreakBench standard prompt with dual-judge agreement.

**Caveat for downstream use as a "benign vs. adversarial" benchmark:** the surface-text distributions of the two subsets are linearly separable to AUC ≈ 1.00 by bag-of-words on user-turn text alone. Any classifier trained on this distinction will trivially saturate; do not interpret high AUC as evidence of internal "intent detection." See `INTENT_OBLITERATION.md` for the full analysis.
